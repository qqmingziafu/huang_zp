Design Process

双钻模型驱动的设计工作流

Phase 01: Discover & Define

需求分析与定义

深入业务一线，通过用户访谈与竞品分析（如 Zendesk、容联七陌），梳理智能客服与 CRM 核心业务逻辑，明确效率提升与转化增长的设计机会点。

# 业务闭环梳理 # 竞品分析 # 痛点挖掘

Phase 02: Structure & Flow

交互架构设计

基于复杂的 AI 问答接待与人工工单流转场景，梳理信息架构 (IA) 与任务流程图。输出低保真原型，确立包含客服工作台在内的整体交互框架。

# 信息架构 (IA) # 任务流程图 # Wireframe

Phase 03: Visual & System

视觉与规范构建

建立适用于 SaaS 系统的原子化设计系统 (Design System)，统一 B 端组件库与视觉语言规范。结合 AI 品牌色，输出最终的高保真 UI 界面。

# UI 设计 # B端组件库 # 高保真落地

Phase 04: Deliver & Validate

交付与还原验收

输出标准化的切图、资产与开发标注。配合前端研发进行 UI 高还原度走查，并开展核心链路的可用性测试，确保设计方案完美落地。

# 设计交付 # UI 走查 # 可用性测试
